About the attempt to compile the kernel for the Motorola PenangF device and why it’s not worth doing.

Thanks FuckYouMoto team.

My experience compiling the kernel for the Motorola PenangF device began with a discussion in the Diablo chat, where this device was mentioned. As someone who had never done this before, I decided to give it a try and took on the challenge. In response, I was provided with the kernel source code, compilation instructions, and a list of known issues and bugs encountered by other users. The list included problems with missing or non-working scripts and build process bugs. Work on the project started late at night. I used Clang and standard tools for compilation but ran into many difficulties. One of the biggest problems was the lack of necessary scripts in the kernel sources, and the official instructions did not work. The scripts mentioned were not included in the sources, the instructions contained many typos, and referenced files that Motorola simply did not provide. Despite attempts to fix this, the problem remained unresolved. Additional difficulties included manually cloning individual kernel modules. When attempting compilation, an error occurred that could not be resolved, preventing successful compilation and device boot after flashing. Despite numerous attempts and fixes, only one out of five participants successfully compiled a working kernel, including myself. Even then, the underlying problem remained unresolved. As I investigated further, it became clear that the source code was incomplete and contained many flaws. Attempts to replace problematic sources with more stable versions from android-13-release-ttt did not help. On the contrary, the build started failing even faster. The person who managed to build the kernel successfully refused to share details of their success, despite claiming they made no changes. This claim cannot be verified, as it was evident from the start that the solution required additional code modifications to even start the compilation process. Based on this experience, I concluded that the Motorola PenangF device, like many other Motorola devices, is not worth purchasing. For a detailed explanation of the reasons, I recommend watching a video prepared by Diablo. Acknowledgments: Diablo — for inspiration and help during the process. r0ddty, shomykohai — for initial issue descriptions and GitHub support. UPDATE 30.08.25 Nothing to explain here; just pasting the issue and translation — After I completely rewrote your script from scratch, I somehow managed to compile the kernel, manually cloning all kernel modules and configuring various things. Even after this, the kernel does not boot (confirmed). Correct instructions for using the toolchain were also missing, so I had to figure it out myself. Various make commands in the script are called without the -j argument, making the build extremely slow and not utilizing the full power of the build machine. In addition, you reference downloading Android13-5.10 GKI at the top, although the phone uses Android12-5.10 GKI. It’s unclear why the GKI should be downloaded at all in the script. The kernel module for Bluetooth made compilation impossible without additional fixes on my side. At least after I fixed all of this, the script worked, but the sources are still broken, so we return to the original problem.
